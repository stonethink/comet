# beta.7 README Native Eval 首屏展示设计

## 目标

在中英文 README 的 `0.4.0-beta.7` 首条发布展示中，突出 Native 相比 0.4.0 Classic 在对齐 Eval 中的效率与稳定性优势，让读者无需下翻即可看到最重要的数据。

## 数据口径

- 来源：`website/zh/eval/comet-native-vs-040-experiment.mdx` 及其英文对应页。
- 实验范围：相同的 16 个业务任务，每个 workflow 每任务运行 3 次，共 96 次运行。
- 效率指标只使用 Native 与 Classic 都严格通过的 41 组相同任务重复，避免把提前失败误计为低消耗。
- 总 Token 包含缓存读取；耗时来自累计模型耗时。由于两组运行窗口不同，耗时只表述为对齐实验中的方向性结果，不写成严格因果结论。

## 首屏展示

保留现有 `[!IMPORTANT]` 发布块，把 beta.7 的一句介绍扩展为一个短标题、四项加粗指标和一个报告链接：

- **总 Token -76.8%**：891,631 vs 3,840,975。
- **Agent 轮次 -57.4%**：39.02 vs 91.66。
- **累计模型耗时 -47.4%**：180.8s vs 343.9s。
- **pass^3 +12.5 个百分点**：87.5% vs 75%，同时两者 pass@3 均为 100%。

中文与英文 README 使用同一结构和同一数字，分别链接到网站的中文、英文完整实验页。文案继续说明 Native 面向强模型，避免暗示 Classic 已被取代。
