# Comet Any Skill Bundle Creator 实现计划

> **致 agentic worker：** 必需子技能：使用 `superpowers:subagent-driven-development`（推荐）或 `superpowers:executing-plans` 按任务逐步实现本计划。步骤使用复选框（`- [ ]`）语法进行跟踪。

**目标：** 构建 `/comet-any` 以及一套确定性的 Bundle 工具链，用于在 Comet 支持的所有平台上创建、校验、评估、发布和分发平台无关的多 Skill Bundle。

**架构：** 新增一个聚焦的 `src/bundle/` 领域，包含严格的 manifest 加载器、内容哈希、编译器 IR、创作状态、Eval 证据、发布和分发服务。宿主 Agent 和 `skill-creator` 负责生成内容，Comet CLI 负责确定性编译、安全检查、生命周期门禁、平台能力报告和原生分发；可选的 Engine 元数据复用 Plan 3 的 Skill Package 加载器，但不会成为运行时强制依赖。

**技术栈：** TypeScript ESM、Node.js 20+、Commander、YAML、Vitest、现有 Comet 平台注册表/拷贝辅助函数、现有 Skill Engine 校验、Markdown Skills。

---

## 文件清单

创建以下聚焦模块：

- `src/bundle/types.ts`：Bundle manifest、编译器 IR、能力、生命周期、Eval 和分发类型。
- `src/bundle/load.ts`：YAML 加载、规范化、locale 覆盖解析、路径包含校验。
- `src/bundle/validate.ts`：语义校验与安全资源图检查。
- `src/bundle/hash.ts`：确定性全 Bundle 内容哈希。
- `src/bundle/compiler.ts`：规范化 IR 与平台无关的安装计划。
- `src/bundle/platform.ts`：将编译器 IR 桥接到现有 Comet 平台注册表和安装位置。
- `src/bundle/preferences.ts`：`.comet/skills.txt` 解析。
- `src/bundle/candidates.ts`：平台 Skill 扫描、实现内容读取、歧义上报。
- `src/bundle/state.ts`：原子化创作状态持久化与漂移调和。
- `src/bundle/draft.ts`：create/optimize 草稿工作流。
- `src/bundle/eval.ts`：quick/full 请求估算与结构化结果校验。
- `src/bundle/publish.ts`：评审门禁与原子化项目发布。
- `src/bundle/distribute.ts`：只读跨平台编译与安装。
- `src/commands/bundle.ts`：CLI 用例编排与稳定的 JSON/文本输出。
- `src/core/bundle-platform.ts`：对现有 29 平台注册表、路径、规则格式、Hook 格式和非破坏性配置合并的窄适配器。

修改以下集成点：

- `src/cli/index.ts`：注册 `comet bundle`。
- `src/core/skills.ts`：暴露可复用的通用 Skill/规则/资源文件规划逻辑，但不改变当前 Comet init 行为。
- `assets/manifest.json`：发布 `/comet-any`。
- `README.md`、`README-zh.md`、`CHANGELOG.md`：记录 Bundle 创作与分发。

在 `test/ts/` 下添加测试，在 `scripts/` 下添加确定性兼容性基准。

```ts
export type BundleSkillVisibility = 'entry' | 'internal';
export type BundleCapability = 'skills' | 'rules' | 'hooks' | 'scripts' | 'references' | 'assets';
export type BundleSideEffect = 'none' | 'read' | 'write' | 'external';

export interface BundleSkillDefinition {
  id: string;
  path: string;
  visibility: BundleSkillVisibility;
}

export interface BundleRuleDefinition {
  id: string;
  path: string;
  mode: 'always' | 'matched';
  match?: string[];
  priority?: number;
  required: boolean;
}

export interface BundleHookDefinition {
  id: string;
  path: string;
}

export interface BundleScriptDefinition {
  id: string;
  path: string;
  sideEffect: BundleSideEffect;
  runtime: 'node' | 'bash' | 'python';
  requiresConfirmation?: boolean;
}

export interface BundlePlatformOverride {
  platform: string;
  replaces: string;
  path: string;
}

export interface BundleManifest {
  apiVersion: 'comet/v1alpha1';
  kind: 'SkillBundle';
  metadata: {
    name: string;
    version: string;
    description: string;
    defaultLocale: string;
    locales: string[];
  };
  skills: BundleSkillDefinition[];
  resources: {
    rules: BundleRuleDefinition[];
    hooks: BundleHookDefinition[];
    references: string[];
    scripts: BundleScriptDefinition[];
    assets: string[];
  };
  platforms: {
    requires: BundleCapability[];
    optional: BundleCapability[];
    overrides: BundlePlatformOverride[];
  };
  engine: { enabled: boolean; path?: string };
}

export interface SkillBundle {
  root: string;
  manifest: BundleManifest;
}
```

- [ ] **Step 1：编写失败的语义校验测试**

- 至少需要一个 `entry` Skill；
- Skill id 与 path 唯一；
